# AGENTS

本文件定義 AI agent 在本專案中的工作規則。除非使用者明確覆寫，所有任務都應遵守以下規則。

## 專案目標

協助製作一份可在瀏覽器中展示的 HTML 簡報。工作應優先支援內容整理、版面設計、互動效果、瀏覽器預覽與最終輸出。

## 工作原則

- 先釐清任務目標、限制與成功條件，再修改檔案。
- 如果需求有多種解讀，先說明目前理解；缺少關鍵資訊時先詢問。
- 採用能解決當前問題的最小改動。
- 只修改與任務直接相關的檔案與區塊。
- 修改前先閱讀相關上下文、呼叫者、共用工具、既有測試與鄰近風格。
- 遵守專案既有命名、檔案結構、錯誤處理、測試與格式化慣例。
- 不覆蓋、不回復、不搬移使用者既有檔案，除非使用者明確要求。
- 若發現互相衝突的寫法或風格，先揭露衝突並說明選擇理由。

## 檔案與輸出規則

- 原始資料放在 `materials/`。
- 可重用或整理後素材放在 `assets/`。
- 最終 HTML 簡報與交付版本放在 `outputs/`。
- 版本需要保留時，輸出檔名使用日期或清楚版本標記。
- 不新增不必要的框架、工具、設定或複雜流程。
- 不順手重構、重新排版、改註解、改命名或整理無關程式碼。

## 驗證規則

- 每個任務都應有明確完成標準。
- 完成後說明已做的驗證；若無法驗證，明確說出原因。
- 測試應驗證業務意圖或重要行為，不只驗證實作細節。
- 不說「測試通過」除非測試確實執行且通過。
- 不說「完成」除非任務真的完成並已驗證。

## 多步驟工作

- 大型任務拆成可檢查的小步驟。
- 每完成重要階段，簡短回報已完成事項、已驗證事項、下一步與風險。
- 如果任務過大、上下文不足或狀態不穩定，先整理進度並請求確認。

## 完成回報格式

完成任務時，簡短回報：

1. 修改了什麼
2. 如何驗證
3. 尚有哪些風險或未完成事項
